Insecure Password Hashing

Passwords should never be stored in plaintext. They should be hashed with a strong algo and salted. If they aren't, the
adversary can use rainbow tables and other cracking mechanisms to easily get the initial password.

Instructions

Navigate to the flask-backend directory and inspect the init_db.py file. You will notice that the password is hashed using MD5.
This is a huge issue as MD5 is deprecated and trivial to crack. In VSCode, install the SQLite Viewer extension and inspect the
database.sqlite file. Copy the 32 bytes under the password column and plug it into CrackStation. What is the initial password?

Remediation

Replace the MD5 hashing with a strong and secure hashing algorithm like bcrypt or argon. Once working, plug the hash into
CrackStation again and see what happens.
