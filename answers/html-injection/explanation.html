HTML Injection Explanation

HTML injection occurs when a website dynamically renders user input without sanitizing it. While not a serious
vulnerability, HTML injection allows the adversary to modify the webpage and potentially expose session cookies or
other sensitive information.

Remediation

On line 4 of html-injection-answer.js, instead of changing the innerHtml property, we change the textContent property.
The textContent property is not parsed as HTML, so its value is displayed as text instead of raw HTML. We could also
strip the HTML tags from the user input, but using the textContent property to dynamically render HTML is easier.

Insecure Code

      const displayName = () => {
        const inputElValue = document.getElementById("name").value;
        const nameDisplayEl = document.getElementById("name-display");
        nameDisplayEl.innerHtml = "hi, " + inputElValue;
      };

Secure Code

      const displayName = () => {
        const inputElValue = document.getElementById("name").value;
        const nameDisplayEl = document.getElementById("name-display");
        nameDisplayEl.textContent = "hi, " + inputElValue;
      };
